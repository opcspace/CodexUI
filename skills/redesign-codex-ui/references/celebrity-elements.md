# Celebrity Element Library

Compose celebrity themes from career-aware elements rather than adding random stars and portraits.

## Select an identity system

- **Actor or filmmaker:** contact sheets, screenplay marks, clapboard metadata, premiere ticket, film-strip navigation, scene numbers, award-season restraint.
- **Musician or idol:** set list, ticket stub, vinyl label, waveform, tour patch, fan-light color, backstage pass, release archive.
- **Athlete:** jersey number, match card, season timeline, scoreboard rhythm, training log, achievement badge.
- **Fashion or photography figure:** magazine folio, crop marks, contact sheet, caption rules, editorial masthead, garment tag.
- **Creator or public intellectual:** monogram, annotated manuscript, notebook tabs, quotation source cards, publication timeline.
- **Fictional or virtual figure:** character status, lore-safe chapter markers, quest log, sprite, constellation, collectible card.

Use one primary career system and one secondary memorabilia family at most.

## Compose the interface

Choose from these functional placements:

| Codex zone | Celebrity element | Functional mapping |
| --- | --- | --- |
| Edition header | Tour, film, season, issue, or fan-club label | Current theme and identity |
| Hero | Licensed portrait or original character artwork | Welcome and primary task framing |
| Navigation | Filmography, discography, chapters, seasons, collections | Real projects and task groups |
| Quick actions | Set list, scene, quest, training, manuscript verbs | Explore, build, review, fix |
| Composer | Letter, control deck, script page, mixing console | Real Codex input |
| Status | Ticket, pass, badge, chart, collection count | Real progress and task state |
| Empty state | Archive card, fan letter, scene slate | Recovery guidance and next action |

## Identity media slots

Support portrait, avatar, signature or wordmark, edition mark, memorabilia image, secondary photo, and optional background texture. Define focal point, fallback, alt text, license, attribution, and commercial-use status for each slot.

Do not use unofficial signatures as authentic, invent quotes, display false awards, imply endorsement, or label a fan theme “official.” Use “inspired edition,” “personal skin,” or “fan-made concept” when appropriate.

## Layering rules

- Keep the portrait as the primary identity anchor.
- Use one large memorabilia object or two small ones per major surface.
- Keep signatures away from functional labels and form fields.
- Keep decorative media behind or outside code, diff, approval, error, and composer content.
- Reduce memorabilia by at least half on narrow screens.
- Prefer authentic metadata supplied by the user; otherwise use neutral placeholders.

## Interaction ideas

Use restrained, optional motion: contact-sheet reveal, record rotation only while loading, stage-light sweep on entry, ticket lift on hover, constellation draw, or card flip for secondary metadata. Disable continuous motion and respect reduced motion.
